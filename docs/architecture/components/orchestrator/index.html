Orchestrator

The Orchestrator is the coordination center of Defog v2. It receives events from the Monitor, consults the Scheduler for decisions, commands the Executor to take actions, and communicates with peer clusters when necessary.

Overview​

Purpose​

The Orchestrator is the only stateful service in Defog v2, responsible for:

  1. Event Processing - Handle stress and RPS events
  2. State Management - Track services, traffic splits, deployments
  3. Decision Coordination - Consult Scheduler and execute actions
  4. Peer Communication - Coordinate with remote clusters
  5. Strategy Execution - Implement both stress and RPS strategies

Event Processing​

Main Event Loop​

Stress Event Handler​

func (o *Orchestrator) handleStress(ctx context.Context, event StressEvent) {
    log.Printf("STRESS: %s has %.2f CPU stress", event.ServiceName, event.CPUStress)
    
    // 1. Update service state
    o.mu.Lock()
    if state, exists := o.localServices[event.ServiceName]; exists {
        state.CurrentStress = event.CPUStress
        state.LastStressCheck = time.Now()
    } else {
        o.localServices[event.ServiceName] = &ServiceState{
            Name:          event.ServiceName,
            Namespace:     event.Namespace,
            CurrentStress: event.CPUStress,
        }
    }
    o.mu.Unlock()
    
    // 2. Calculate traffic split via Scheduler
    decision := o.schedulerClient.CalculateSplit(ctx, event.CPUStress)
    
    // 3. Execute traffic split
    o.adjustTrafficSplit(ctx, event.ServiceName, event.Namespace, decision.MirrorPercent)
}

RPS Event Handler​

func (o *Orchestrator) handleHighRPS(ctx context.Context, event HighRPSEvent) {
    log.Printf("HIGH RPS: %s (mirrored) has %.2f RPS", event.ServiceName, event.CurrentRPS)
    
    // Only act on mirrored services
    if event.IsLocal {
        return
    }
    
    // 1. Check if already deployed locally
    if o.hasLocalDeployment(event.ServiceName) {
        log.Printf("Service %s already local, considering scaling", event.ServiceName)
        o.considerScaling(ctx, event.ServiceName, event.CurrentRPS)
        return
    }
    
    // 2. Check local capacity
    capacity := o.getLocalCapacity(ctx)
    
    if o.hasEnoughCapacity(capacity) {
        // Deploy locally
        o.deployServiceLocally(ctx, event.ServiceName, event.Namespace)
        return
    }
    
    // 3. Try to make space via LFU
    decision := o.tryMakeSpace(ctx, event.ServiceName)
    
    if decision.CanMakeSpace {
        // Evict low RPS service, then deploy
        for _, svc := range decision.ServicesToDelete {
            o.deleteServiceLocally(ctx, svc, event.Namespace)
        }
        o.deployServiceLocally(ctx, event.ServiceName, event.Namespace)
        return
    }
    
    // 4. Cannot make space - ask peers
    o.askPeerToDeploy(ctx, event.ServiceName, event.Namespace)
}

Peer Communication​

Querying Peer Capacity​

Requesting Deployment​

func (o *Orchestrator) askPeerToDeploy(ctx context.Context, serviceName, namespace string) {
    log.Printf("Asking peers to deploy %s", serviceName)
    
    rpsMetrics := o.getRPSForService(serviceName)
    
    req := CanDeployRequest{
        ServiceName:   serviceName,
        Namespace:     namespace,
        SourceCluster: o.config.ClusterID,
        CurrentRPS:    rpsMetrics.CurrentRPS,
        Resources:     o.getServiceResources(serviceName),
    }
    
    // Try peers in order of preference
    for clusterID, peerClient := range o.peerClients {
        resp, err := peerClient.CanDeploy(ctx, req)
        if err != nil {
            log.Printf("Failed to ask %s: %v", clusterID, err)
            continue
        }
        
        switch resp.Status {
        case "already_have_it":
            log.Printf("%s already has %s deployed", clusterID, serviceName)
            o.updateTrafficRouting(ctx, serviceName, namespace, clusterID)
            return
            
        case "will_deploy":
            log.Printf("%s will deploy %s", clusterID, serviceName)
            // Wait for confirmation, then update traffic
            o.waitForPeerDeployment(ctx, clusterID, serviceName)
            o.updateTrafficRouting(ctx, serviceName, namespace, clusterID)
            return
            
        case "cannot_deploy":
            log.Printf("%s cannot deploy: %s", clusterID, resp.Reason)
            continue
        }
    }
    
    log.Printf("No peer cluster can deploy %s", serviceName)
}

Traffic Management​

Adjusting Traffic Splits​

func (o *Orchestrator) adjustTrafficSplit(ctx context.Context, serviceName, namespace string, mirrorPercent int) {
    localPercent := 100 - mirrorPercent
    
    log.Printf("Adjusting traffic for %s: %d%% local, %d%% mirror", 
        serviceName, localPercent, mirrorPercent)
    
    req := TrafficSplitRequest{
        ServiceName: serviceName,
        Namespace:   namespace,
        Backends: []TrafficBackend{
            {Service: serviceName, Weight: localPercent},
            {Service: serviceName + "-mirrored", Weight: mirrorPercent},
        },
    }
    
    if err := o.executorClient.UpdateTrafficSplit(ctx, req); err != nil {
        log.Printf("Failed to update traffic split: %v", err)
        return
    }
    
    // Update state
    o.mu.Lock()
    if state, exists := o.localServices[serviceName]; exists {
        state.TrafficSplitLocal = localPercent
    }
    o.mu.Unlock()
    
    log.Printf("Traffic split updated successfully")
}

Gradual Traffic Shifting​

func (o *Orchestrator) gradualShift(ctx context.Context, serviceName, namespace string, targetPercent int) {
    currentState := o.getServiceState(serviceName)
    currentPercent := currentState.TrafficSplitLocal
    
    increment := o.config.TrafficSplitIncrement  // e.g., 10%
    
    // Gradually shift traffic
    for current := currentPercent; current != targetPercent; {
        if targetPercent > current {
            current = min(current + increment, targetPercent)
        } else {
            current = max(current - increment, targetPercent)
        }
        
        mirrorPercent := 100 - current
        o.adjustTrafficSplit(ctx, serviceName, namespace, mirrorPercent)
        
        // Wait between shifts
        time.Sleep(o.config.TrafficShiftDelay)
    }
}

Scaling​

Important: The Orchestrator should NOT be scaled horizontally due to stateful nature. For high availability:

  1. Use replicas: 1 with leader election (future)
  2. Or use active-passive setup
  3. State can be externalized to database (future)

Troubleshooting​

Issue: Events Not Processing​

Check event queue:

kubectl exec -n defog-system deployment/orchestrator -- \
  curl localhost:8080/state | jq '.queueSize'

Check logs:

kubectl logs -n defog-system deployment/orchestrator | grep "ERROR"

Issue: Peer Communication Failing​

Test connectivity:

kubectl exec -it -n defog-system deployment/orchestrator -- \
  curl -v https://cluster-b-executor.example.com/peer/v1/health

Check peer state:

kubectl exec -n defog-system deployment/orchestrator -- \
  curl localhost:8080/state | jq '.peerClusters'

Best Practices​

  1. Run single replica - Orchestrator is stateful
  2. Monitor event queue - Alert if backing up
  3. Set peer timeouts - Prevent hanging requests
  4. Log all decisions - For debugging and auditing
  5. Use circuit breakers - For peer communication
  6. Health check peers - Regularly verify connectivity
