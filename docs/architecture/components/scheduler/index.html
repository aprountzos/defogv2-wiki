Scheduler

The Scheduler is a stateless decision engine that implements pluggable algorithms for service placement, eviction, and cluster scoring. It has no side effects—it only returns decisions based on inputs.

Overview​

Purpose​

The Scheduler provides pure business logic without side effects:

  1. Stress-Based Splitting - Calculate traffic split percentages
  2. RPS-Based LFU - Determine which services to evict
  3. Cluster Scoring - Rank peer clusters for placement
  4. Placement Decisions - Decide where to deploy services
  5. Pluggable Algorithms - Easy to add new strategies

Key Characteristics​

Stateless​

  • No internal state
  • No database or cache
  • Same input = same output
  • Easy to test and reason about

Side-Effect Free​

The Scheduler never:

  • Makes API calls
  • Modifies resources
  • Stores data
  • Emits events

It only:

  • Receives requests
  • Runs algorithms
  • Returns decisions

Algorithms​

1. Stress-Based Split Algorithm​

Calculates traffic split based on pod stress level.

Implementation:

type StressBasedSplitAlgorithm struct {
    thresholds map[string]float64
}

func (a *StressBasedSplitAlgorithm) CalculateSplit(stress float64) int {
    if stress < 0.70 {
        return 0  // 0% to mirror
    } else if stress < 0.80 {
        return 10  // 10% to mirror
    } else if stress < 0.90 {
        return 30  // 30% to mirror
    } else {
        return 50  // 50% to mirror
    }
}

Example:

// Input
{
  "serviceName": "api",
  "currentStress": 0.85
}

// Output
{
  "action": "shift_traffic",
  "mirrorPercent": 30,
  "localPercent": 70,
  "reason": "stress between 0.80 and 0.90"
}

2. RPS-Based LFU Algorithm​

Determines which service to evict based on RPS (Least Frequently Used).

Implementation:

type RPSBasedLFU struct {
    minRPSToKeep float64
}

func (l *RPSBasedLFU) DecideEviction(req EvictionRequest) EvictionDecision {
    // Filter only local services
    var localServices []ServiceRPSMetrics
    for _, svc := range req.CurrentServices {
        if svc.IsLocal {
            localServices = append(localServices, svc)
        }
    }
    
    if len(localServices) == 0 {
        return EvictionDecision{
            CanMakeSpace: false,
            Reason:       "no local services to evict",
        }
    }
    
    // Sort by RPS (ascending - lowest first)
    sort.Slice(localServices, func(i, j int) bool {
        return localServices[i].CurrentRPS < localServices[j].CurrentRPS
    })
    
    // Check if lowest RPS service is below minRPSToKeep
    lowestRPS := localServices[0].CurrentRPS
    
    if lowestRPS >= l.minRPSToKeep {
        return EvictionDecision{
            CanMakeSpace: false,
            Reason:       "all services above minRPSToKeep threshold",
            LowestRPS:    lowestRPS,
        }
    }
    
    return EvictionDecision{
        CanMakeSpace:     true,
        ServicesToDelete: []string{localServices[0].ServiceName},
        Reason:           "service has low RPS and below threshold",
        LowestRPS:        lowestRPS,
    }
}

Example:

// Input
{
  "currentServices": [
    {"serviceName": "api", "isLocal": true, "currentRps": 450},
    {"serviceName": "payments", "isLocal": true, "currentRps": 320},
    {"serviceName": "old-api", "isLocal": true, "currentRps": 8},
    {"serviceName": "reports", "isLocal": true, "currentRps": 150}
  ],
  "minRpsToKeep": 50.0
}

// Output
{
  "canMakeSpace": true,
  "servicesToDelete": ["old-api"],
  "reason": "service has low RPS (8) and below threshold (50)",
  "lowestRps": 8.0
}

3. Weighted Scoring Algorithm​

Scores peer clusters for placement decisions.

Scoring Formula:

Score = (CPU_Score * 0.4) + (Memory_Score * 0.4) + (Latency_Score * 0.2)

CPU_Score = FreeCPU / TotalCPU
Memory_Score = FreeMemory / TotalMemory

Latency_Score:
  < 10ms   → 1.0
  < 50ms   → 0.8
  < 100ms  → 0.5
  >= 100ms → 0.2

Implementation:

type WeightedScoringAlgorithm struct {
    cpuWeight     float64
    memoryWeight  float64
    latencyWeight float64
}

func (a *WeightedScoringAlgorithm) ScoreCluster(cluster ClusterInfo) float64 {
    // Calculate CPU score
    cpuScore := cluster.Capacity.FreeCPU / cluster.Capacity.TotalCPU
    
    // Calculate memory score
    memScore := cluster.Capacity.FreeMemory / cluster.Capacity.TotalMemory
    
    // Calculate latency score
    latencyScore := 1.0
    if cluster.Latency != nil {
        latencyMs := float64(cluster.Latency.Milliseconds())
        if latencyMs < 10 {
            latencyScore = 1.0
        } else if latencyMs < 50 {
            latencyScore = 0.8
        } else if latencyMs < 100 {
            latencyScore = 0.5
        } else {
            latencyScore = 0.2
        }
    }
    
    // Weighted sum
    return (cpuScore * a.cpuWeight) +
           (memScore * a.memoryWeight) +
           (latencyScore * a.latencyWeight)
}

Example:

// Input
{
  "clusters": [
    {
      "clusterId": "cluster-b",
      "capacity": {
        "freeCpu": 4.5,
        "totalCpu": 8.0,
        "freeMemory": 8.0,
        "totalMemory": 16.0
      },
      "latency": 15000000  // 15ms
    },
    {
      "clusterId": "cluster-c",
      "capacity": {
        "freeCpu": 2.0,
        "totalCpu": 8.0,
        "freeMemory": 10.0,
        "totalMemory": 16.0
      },
      "latency": 80000000  // 80ms
    }
  ]
}

// Output
{
  "scores": {
    "cluster-b": 0.84,  // High score (good capacity + low latency)
    "cluster-c": 0.56   // Lower score (less CPU + higher latency)
  },
  "recommended": "cluster-b"
}

API Endpoints​

POST /placement/decide​

Get placement decision for a service.

Request:

{
  "serviceName": "payments",
  "namespace": "default",
  "resources": {
    "cpuRequest": "500m",
    "memoryRequest": "512Mi"
  },
  "currentStress": 0.85,
  "availableClusters": [
    {
      "clusterId": "cluster-b",
      "capacity": {
        "freeCpu": 4.5,
        "freeMemory": 8.0
      },
      "healthy": true
    }
  ]
}

Response:

{
  "action": "deploy_remote",
  "targetCluster": "cluster-b",
  "reason": "local stress high, peer has capacity",
  "recommendedReplicas": 2
}

Action Values:

  • deploy_local - Deploy on local cluster
  • deploy_remote - Deploy on peer cluster
  • scale_local - Scale existing local deployment
  • no_action - Do nothing

Testing​

Unit Test Example​

func TestLFUEviction(t *testing.T) {
    lfu := NewRPSBasedLFU(50.0)  // minRPSToKeep
    
    req := EvictionRequest{
        CurrentServices: []ServiceRPSMetrics{
            {ServiceName: "api", IsLocal: true, CurrentRPS: 450},
            {ServiceName: "old-api", IsLocal: true, CurrentRPS: 8},
            {ServiceName: "payments", IsLocal: true, CurrentRPS: 320},
        },
        MinRPSToKeep: 50.0,
    }
    
    decision := lfu.DecideEviction(req)
    
    assert.True(t, decision.CanMakeSpace)
    assert.Equal(t, []string{"old-api"}, decision.ServicesToDelete)
    assert.Equal(t, 8.0, decision.LowestRPS)
}

Integration Test Example​

func TestPlacementDecision(t *testing.T) {

Adding Custom Algorithms​

1. Define Interface​

type PlacementAlgorithm interface {
    Decide(ctx context.Context, req PlacementRequest) PlacementDecision
}

2. Implement Algorithm​

type CostBasedAlgorithm struct {
    costPerCPUHour float64
}

func (a *CostBasedAlgorithm) Decide(ctx context.Context, req PlacementRequest) PlacementDecision {
    // Find cheapest cluster with capacity
    var cheapest ClusterInfo
    minCost := math.MaxFloat64
    
    for _, cluster := range req.AvailableClusters {
        cost := a.calculateCost(cluster)
        if cost < minCost && cluster.Capacity.FreeCPU > 1.0 {
            minCost = cost
            cheapest = cluster
        }
    }
    
    return PlacementDecision{
        Action:        "deploy_remote",
        TargetCluster: cheapest.ClusterID,
        Reason:        fmt.Sprintf("lowest cost: $%.2f/hour", minCost),
    }
}

3. Register Algorithm​

scheduler.RegisterAlgorithm("cost", &CostBasedAlgorithm{
    costPerCPUHour: 0.05,
})

4. Configure​

PLACEMENT_ALGORITHM: "cost"
COST_PER_CPU_HOUR: "0.05"

Best Practices​

  1. Keep algorithms pure - No side effects
  2. Make decisions deterministic - Same input = same output
  3. Log all decisions - For debugging and auditing
  4. Test thoroughly - Unit test each algorithm
  5. Document thresholds - Explain why values chosen
  6. Make pluggable - Easy to swap algorithms

Troubleshooting​

Issue: Wrong Decisions​

Check algorithm logic:

# Enable debug logging
kubectl set env deployment/scheduler LOG_LEVEL=debug -n defog-system

# Check decision logs
kubectl logs -n defog-system deployment/scheduler | grep "Decision"

Issue: Slow Response​

Check CPU usage:

kubectl top pod -n defog-system -l app=scheduler

Scale if needed:

kubectl scale deployment/scheduler --replicas=3 -n defog-system
