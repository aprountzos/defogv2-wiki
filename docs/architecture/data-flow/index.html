Flow 1: Stress-Based Traffic Shift​

Scenario: Local pods become overloaded, need immediate relief.

Sequence Diagram​

Timeline​

TimeEventActionResult
T+0sCPU reaches 85%Monitor detects stressStressEvent emitted
T+2sOrchestrator receives eventCalculate split: 30%Decision made
T+5sExecutor updates CRDApply TrafficSplit30% traffic to mirror
T+10sLoad reduces to 60%Monitor detects recoveryStressEvent emitted
T+12sOrchestrator receives eventCalculate split: 0%Decision made
T+15sExecutor updates CRDApply TrafficSplit100% traffic to local

Configuration Impact​

stressThresholds:
  medium: 0.70   # At 85%, this triggers high threshold
  high: 0.80     # ← Triggered here
  critical: 0.90

trafficSplitIncrement: 10  # Could adjust by 10% increments
