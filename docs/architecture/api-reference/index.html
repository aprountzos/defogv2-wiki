API Reference

Complete API documentation for internal services and peer-to-peer communication.

Peer API (Executor)​

The Executor service exposes these endpoints for remote cluster communication.

Authentication​

All peer API requests require Bearer token authentication:

Authorization: Bearer <cluster-token>

Endpoints​

GET /peer/v1/capacity​

Get current cluster capacity.

Response:

{
  "clusterId": "cluster-a",
  "freeCpu": 4.5,
  "freeMemory": 8.2,
  "totalCpu": 8.0,
  "totalMemory": 16.0,
  "schedulablePods": 50,
  "timestamp": "2024-01-15T10:30:00Z"
}

Response Codes:

  • 200 OK - Success
  • 401 Unauthorized - Invalid or missing token
  • 503 Service Unavailable - Cannot calculate capacity

POST /peer/v1/can-deploy​

Check if cluster can deploy a service.

Request:

{
  "serviceName": "payments",
  "namespace": "default",
  "resources": {
    "cpuRequest": "500m",
    "memoryRequest": "512Mi",
    "cpuLimit": "1000m",
    "memoryLimit": "1Gi"
  },
  "sourceCluster": "cluster-b",
  "currentRps": 450.0
}

Response:

{
  "status": "will_deploy",
  "message": "Deployment will be initiated",
  "reason": "has_capacity"
}

Status Values:

  • already_have_it - Service already deployed locally
  • will_deploy - Will deploy the service
  • cannot_deploy - Cannot deploy (see reason)

Response Codes:

  • 200 OK - Decision made
  • 400 Bad Request - Invalid request
  • 401 Unauthorized - Invalid token

POST /peer/v1/deploy​

Deploy a service on this cluster.

Request:

{
  "serviceName": "payments",
  "namespace": "default",
  "image": "myapp:v1.0.0",
  "replicas": 2,
  "resources": {
    "cpuRequest": "500m",
    "memoryRequest": "512Mi",
    "cpuLimit": "1000m",
    "memoryLimit": "1Gi"
  },
  "labels": {
    "app": "payments",
    "version": "v1"
  },
  "sourceCluster": "cluster-b"
}

Response:

{
  "success": true,
  "serviceName": "payments",
  "endpoint": "payments.default.svc.cluster.local",
  "message": "Deployment created successfully"
}

Response Codes:

  • 200 OK - Deployment successful
  • 400 Bad Request - Invalid request
  • 401 Unauthorized - Invalid token
  • 409 Conflict - Service already exists
  • 500 Internal Server Error - Deployment failed

DELETE /peer/v1/services/{name}​

Delete a service deployment.

Path Parameters:

  • name - Service name to delete

Query Parameters:

  • namespace - Namespace (default: "default")

Response:

{
  "status": "deleted",
  "message": "Service payments deleted successfully"
}

Response Codes:

  • 200 OK - Deleted successfully
  • 401 Unauthorized - Invalid token
  • 404 Not Found - Service not found
  • 500 Internal Server Error - Deletion failed

GET /peer/v1/metrics/{service}​

Get RPS metrics for a service.

Path Parameters:

  • service - Service name

Query Parameters:

  • namespace - Namespace (default: "default")

Response:

{
  "serviceName": "payments",
  "namespace": "default",
  "isLocal": true,
  "currentRps": 450.0,
  "avgRps": 420.0,
  "peakRps": 580.0,
  "totalRequests": 1250000,
  "timestamp": "2024-01-15T10:30:00Z"
}

Response Codes:

  • 200 OK - Metrics returned
  • 401 Unauthorized - Invalid token
  • 404 Not Found - Service not found

GET /peer/v1/health​

Health check endpoint.

Response:

{
  "status": "healthy",
  "version": "v2.0.0",
  "clusterId": "cluster-a"
}

Response Codes:

  • 200 OK - Healthy
  • 503 Service Unavailable - Unhealthy

Internal APIs​

Monitor API​

Exposed by the Metrics Monitor service.

GET /metrics/stress​

Get stress metrics for a service.

Query Parameters:

  • service - Service name (required)
  • namespace - Namespace (default: "default")

Response:

{
  "serviceName": "api",
  "namespace": "default",
  "podStress": [
    {
      "podName": "api-pod-1",
      "cpuUsage": 0.85,
      "memoryUsage": 0.70,
      "stressLevel": "high"
    }
  ],
  "avgStress": 0.85,
  "maxStress": 0.92,
  "timestamp": "2024-01-15T10:30:00Z"
}

GET /metrics/rps​

Get RPS metrics for a service.

Query Parameters:

  • service - Service name (required)
  • namespace - Namespace (default: "default")

Response:

{
  "serviceName": "api",
  "namespace": "default",
  "isLocal": true,
  "currentRps": 450.0,
  "avgRps": 420.0,
  "peakRps": 580.0,
  "totalRequests": 1250000,
  "timestamp": "2024-01-15T10:30:00Z"
}

GET /metrics/capacity​

Get cluster capacity report.

Response:

{
  "clusterId": "cluster-a",
  "freeCpu": 4.5,
  "freeMemory": 8.2,
  "totalCpu": 8.0,
  "totalMemory": 16.0,
  "schedulablePods": 50,
  "timestamp": "2024-01-15T10:30:00Z"
}

GET /metrics/snapshot​

Get all metrics in one call.

Response:

{
  "clusterId": "cluster-a",
  "services": {
    "api": {
      "serviceName": "api",
      "isLocal": true,
      "currentRps": 450.0,
      "avgStress": 0.85
    },
    "payments": {
      "serviceName": "payments",
      "isLocal": false,
      "currentRps": 320.0,
      "avgStress": 0.0
    }
  },
  "timestamp": "2024-01-15T10:30:00Z"
}

Scheduler API​

Exposed by the Scheduler service.

POST /placement/decide​

Get placement decision.

Request:

{
  "serviceName": "payments",
  "namespace": "default",
  "resources": {
    "cpuRequest": "500m",
    "memoryRequest": "512Mi"
  },
  "currentStress": 0.85,
  "availableClusters": [
    {
      "clusterId": "cluster-b",
      "apiEndpoint": "https://cluster-b.example.com",
      "capacity": {
        "freeCpu": 4.5,
        "freeMemory": 8.0
      },
      "healthy": true
    }
  ]
}

Response:

{
  "action": "deploy_remote",
  "targetCluster": "cluster-b",
  "reason": "local stress high, peer has capacity",
  "recommendedReplicas": 2
}

Action Values:

  • deploy_local - Deploy on local cluster
  • deploy_remote - Deploy on peer cluster
  • no_action - Do nothing
  • scale_local - Scale existing local deployment

POST /eviction/decide​

Get eviction decision based on LFU.

Request:

{
  "currentServices": [
    {
      "serviceName": "api",
      "isLocal": true,
      "currentRps": 450.0
    },
    {
      "serviceName": "old-api",
      "isLocal": true,
      "currentRps": 8.0
    }
  ],
  "requiredSpace": {
    "cpuRequest": "500m",
    "memoryRequest": "512Mi"
  },
  "minRpsToKeep": 50.0
}

Response:

{
  "canMakeSpace": true,
  "servicesToDelete": ["old-api"],
  "reason": "service has low RPS and below threshold",
  "lowestRps": 8.0
}

POST /clusters/score​

Score peer clusters for placement.

Request:

{
  "clusters": [
    {
      "clusterId": "cluster-b",
      "capacity": {
        "freeCpu": 4.5,
        "freeMemory": 8.0
      },
      "latency": 15000000
    }
  ]
}

Response:

{
  "scores": {
    "cluster-b": {
      "totalScore": 0.85,
      "cpuScore": 0.90,
      "memoryScore": 0.80,
      "latencyScore": 0.85
    }
  }
}

Executor API (Local)​

Exposed by the Executor for local orchestrator.

POST /api/v1/deployments​

Create a deployment.

Request:

{
  "serviceName": "payments",
  "namespace": "default",
  "image": "myapp:v1.0.0",
  "replicas": 2,
  "resources": {
    "cpuRequest": "500m",
    "memoryRequest": "512Mi",
    "cpuLimit": "1000m",
    "memoryLimit": "1Gi"
  }
}

Response:

{
  "success": true,
  "serviceName": "payments",
  "endpoint": "payments.default.svc.cluster.local",
  "message": "Deployment created successfully"
}

DELETE /api/v1/deployments/{name}​

Delete a deployment.

Path Parameters:

  • name - Service name

Response:

{
  "status": "deleted"
}

PATCH /api/v1/deployments/{name}/scale​

Scale a deployment.

Path Parameters:

  • name - Service name

Request:

{
  "replicas": 5
}

Response:

{
  "status": "scaled",
  "replicas": 5
}

POST /api/v1/traffic-splits​

Create or update a TrafficSplit.

Request:

{
  "serviceName": "api",
  "namespace": "default",
  "backends": [
    {
      "service": "api",
      "weight": 70
    },
    {
      "service": "api-mirrored",
      "weight": 30
    }
  ]
}

Response:

{
  "success": true,
  "message": "Traffic split applied successfully"
}

Event Types​

StressEvent​

Emitted when pod stress crosses threshold.

{
  "serviceName": "api",
  "namespace": "default",
  "podName": "api-pod-1",
  "cpuStress": 0.85,
  "memoryStress": 0.70,
  "stressLevel": "high",
  "timestamp": "2024-01-15T10:30:00Z"
}

HighRPSEvent​

Emitted when service RPS exceeds threshold.

{
  "serviceName": "payments-mirrored",
  "namespace": "default",
  "isLocal": false,
  "currentRps": 500.0,
  "threshold": 300.0,
  "timestamp": "2024-01-15T10:30:00Z"
}

LowRPSEvent​

Emitted when service RPS drops below threshold.

{
  "serviceName": "old-api",
  "namespace": "default",
  "isLocal": true,
  "currentRps": 8.0,
  "threshold": 10.0,
  "timestamp": "2024-01-15T10:30:00Z"
}

Error Responses​

All APIs use standard HTTP status codes and return errors in this format:

{
  "error": {
    "code": "INSUFFICIENT_CAPACITY",
    "message": "Not enough CPU to deploy service",
    "details": {
      "required": "2000m",
      "available": "500m"
    }
  }
}

Common Error Codes​

CodeDescription
INVALID_REQUESTRequest validation failed
UNAUTHORIZEDInvalid or missing authentication
NOT_FOUNDResource not found
ALREADY_EXISTSResource already exists
INSUFFICIENT_CAPACITYNot enough cluster resources
DEPLOYMENT_FAILEDKubernetes deployment failed
INTERNAL_ERRORInternal server error

Rate Limiting​

Peer API endpoints are rate limited:

  • Capacity checks: 100 requests/hour per cluster
  • Deployment operations: 50 requests/hour per cluster
  • Metrics queries: 200 requests/hour per cluster

Rate limit headers:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1642251600

Versioning​

API version is included in the URL path:

  • Current version: v1
  • Peer API: /peer/v1/...
  • Internal API: /api/v1/...

Breaking changes will increment the major version.
