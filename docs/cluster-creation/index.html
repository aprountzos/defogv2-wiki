Creating Clusters

Overview​

In this guide, you'll create three K3s clusters using K3d to simulate a cloud-fog-edge architecture:

Cluster characteristics:

ClusterRolePurpose
topCloudCentralized services, databases, heavy processing
midFogRegional processing, data aggregation
botEdgeUser-facing services, low-latency responses

Prerequisites​

Before starting, ensure you have:

  • ✅ Docker running (test with docker ps)
  • ✅ K3d installed (test with k3d version)
  • ✅ kubectl installed (test with kubectl version --client)
  • ✅ At least 8GB RAM available
  • ✅ At least 10GB free disk space

Why This Approach?​

We'll create clusters step-by-step to understand each component. Here's what makes our setup special:

1. Shared Docker Network​

All clusters on one network so they can discover each other:

2. Stable IP Addresses​

K3d assigns IPs dynamically, but we need them to be predictable for certificates. We'll:

  1. Create cluster once to discover IPs
  2. Delete it
  3. Recreate with correct TLS SANs for those IPs

3. Traefik Disabled​

We disable Traefik (K3s default ingress) because:

  • Linkerd provides its own ingress capabilities
  • Reduces resource usage
  • Avoids port conflicts

4. Custom API Ports​

Each cluster gets its own API port on localhost:

  • top: 6443
  • mid: 6444
  • bot: 6445

This lets you manage all clusters from your machine.

Step 1: Create Docker Network​

First, create a shared network for all clusters:

docker network create multicluster-example

Verify:

docker network ls | grep multicluster-example

You should see:

NETWORK ID     NAME                    DRIVER    SCOPE
abc123def456   multicluster-example    bridge    local

Step 2: Create the Top (Cloud) Cluster​

We'll create the top cluster with a two-step process to get stable IPs.

Step 2.1: Initial Creation​

Create the cluster initially without TLS SANs:

k3d cluster create top \
  --api-port 6443 \
  --network multicluster-example \
  --k3s-arg "--cluster-domain=top.k3d.example.com@server:*" \
  --k3s-arg "--disable=traefik@server:*"

What each flag does:

FlagPurpose
--api-port 6443Expose Kubernetes API on port 6443
--network multicluster-exampleUse our shared network
--k3s-arg "--cluster-domain=..."Set custom DNS domain
--k3s-arg "--disable=traefik@server:*"Disable Traefik ingress

Wait for it to be ready:

kubectl cluster-info --context k3d-top

Step 2.2: Discover Assigned IPs​

Find the IPs that Docker assigned:

# Get serverlb IP
docker inspect k3d-top-serverlb \
  --format '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}'

# Get server IP
docker inspect k3d-top-server-0 \
  --format '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}'

Save these IPs! You'll need them in the next step.

Example output:

172.18.0.2  # serverlb
172.18.0.3  # server-0

Step 2.3: Recreate with Correct SANs​

Delete and recreate the cluster with TLS SANs for the discovered IPs:

# Delete temporary cluster
k3d cluster delete top

# Recreate with correct IPs (replace with YOUR IPs from above)
k3d cluster create top \
  --api-port 6443 \
  --network multicluster-example \
  --k3s-arg "--cluster-domain=top.k3d.example.com@server:*" \
  --k3s-arg "--disable=traefik@server:*" \
  --k3s-arg "--tls-san=172.18.0.2@server:*" \
  --k3s-arg "--tls-san=172.18.0.3@server:*"
Replace IPs

Use the IPs you discovered in step 2.2, not the example IPs shown here!

Step 2.4: Verify Cluster​

# Check cluster is running
k3d cluster list

# Check nodes
kubectl get nodes --context k3d-top

# Check system pods
kubectl get pods -n kube-system --context k3d-top

You should see output like:

NAME                      STATUS   ROLES                  AGE   VERSION
k3d-top-server-0          Ready    control-plane,master   30s   v1.28.x

Step 3: Create the Mid (Fog) Cluster​

Repeat the same process for the mid cluster.

Step 3.1: Initial Creation​

k3d cluster create mid \
  --api-port 6444 \
  --network multicluster-example \
  --k3s-arg "--cluster-domain=mid.k3d.example.com@server:*" \
  --k3s-arg "--disable=traefik@server:*"

Step 3.2: Discover IPs​

docker inspect k3d-mid-serverlb \
  --format '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}'

docker inspect k3d-mid-server-0 \
  --format '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}'

Step 3.3: Recreate with SANs​

k3d cluster delete mid

# Replace with YOUR discovered IPs
k3d cluster create mid \
  --api-port 6444 \
  --network multicluster-example \
  --k3s-arg "--cluster-domain=mid.k3d.example.com@server:*" \
  --k3s-arg "--disable=traefik@server:*" \
  --k3s-arg "--tls-san=YOUR_SERVERLB_IP@server:*" \
  --k3s-arg "--tls-san=YOUR_SERVER_IP@server:*"

Step 3.4: Verify​

kubectl get nodes --context k3d-mid

Step 4.4: Verify​

kubectl get nodes --context k3d-bot

Step 5: Verify All Clusters​

Check All Clusters Are Running​

k3d cluster list

Expected output:

NAME   SERVERS   AGENTS   LOADBALANCER
bot    1/1       0/0      true
mid    1/1       0/0      true
top    1/1       0/0      true

Check kubectl Contexts​

kubectl config get-contexts

You should see three contexts:

CURRENT   NAME      CLUSTER   AUTHINFO        NAMESPACE
          k3d-bot   k3d-bot   admin@k3d-bot   default
          k3d-mid   k3d-mid   admin@k3d-mid   default
*         k3d-top   k3d-top   admin@k3d-top   default

Test Each Cluster​

  • Top Cluster
  • Mid Cluster
  • Bot Cluster
kubectl get nodes --context k3d-top
kubectl get pods -A --context k3d-top
kubectl get nodes --context k3d-mid
kubectl get pods -A --context k3d-mid
kubectl get nodes --context k3d-bot
kubectl get pods -A --context k3d-bot

Verify Network Connectivity​

Test that clusters can reach each other on the Docker network:

# From top, ping mid
docker exec k3d-top-server-0 ping -c 3 k3d-mid-server-0

# From mid, ping bot
docker exec k3d-mid-server-0 ping -c 3 k3d-bot-server-0

# From bot, ping top
docker exec k3d-bot-server-0 ping -c 3 k3d-top-server-0

All pings should succeed!

Understanding Your Setup​

Architecture​

Port Mapping​

ClusterAPI Port on HostContainer API PortDocker Network IP

Storage​

Each cluster has its own storage:

# View cluster volumes
docker volume ls | grep k3d

# Inspect a volume
docker volume inspect k3d-top-server-0

Common Operations​

Switching Between Clusters​

# Switch to top
kubectl config use-context k3d-top

# Switch to mid
kubectl config use-context k3d-mid

# Switch to bot
kubectl config use-context k3d-bot

# Check current context
kubectl config current-context

Stopping Clusters​

# Stop one cluster
k3d cluster stop top

# Stop all clusters
k3d cluster stop top mid bot

Note: Stopping preserves all data. Start again with:

k3d cluster start top mid bot

Restarting Clusters​

# Restart all
k3d cluster stop top mid bot
k3d cluster start top mid bot

Viewing Logs​

# K3s server logs
docker logs k3d-top-server-0

# Follow logs
docker logs -f k3d-top-server-0

# Last 50 lines
docker logs --tail 50 k3d-top-server-0

Accessing Cluster Containers​

# Shell into server container
docker exec -it k3d-top-server-0 sh

# Run a command
docker exec k3d-top-server-0 kubectl get nodes

# Check processes
docker exec k3d-top-server-0 ps aux

Troubleshooting​

Cluster Won't Start​

Check Docker is running:

docker ps

Check for port conflicts:

# Check if ports are already in use
lsof -i :6443
lsof -i :6444
lsof -i :6445

Solution: Kill processes using those ports or use different ports.

Can't Connect to Cluster​

Check kubeconfig:

kubectl config view --context k3d-top

Verify API server:

curl -k https://localhost:6443/healthz

Check container is running:

docker ps | grep k3d-top

Clusters Can't Communicate​

Verify network:

docker network inspect multicluster-example

All clusters should be listed under "Containers".

Test connectivity:

docker exec k3d-top-server-0 ping k3d-mid-server-0

Recreate network if needed:

# Stop clusters
k3d cluster stop top mid bot

# Remove old network
docker network rm multicluster-example

# Create new network
docker network create multicluster-example

# Start clusters
k3d cluster start top mid bot

Out of Disk Space​

Check Docker disk usage:

docker system df

Clean up:

# Remove stopped containers
docker container prune

# Remove unused images
docker image prune -a

# Remove unused volumes
docker volume prune

Cluster Feels Slow​

Check resource usage:

docker stats

Allocate more resources to Docker:

  • Docker Desktop → Settings → Resources
  • Increase CPU and Memory allocations

Cleaning Up​

Delete Individual Cluster​

k3d cluster delete top

Delete All Clusters​

k3d cluster delete top mid bot

Complete Cleanup​

# Delete clusters
k3d cluster delete top mid bot

# Remove network
docker network rm multicluster-example

# Remove volumes (CAUTION: destroys all data)
docker volume rm $(docker volume ls -q | grep k3d)

# Clean Docker
docker system prune -a
Warning

Deleting volumes destroys all data in your clusters. Make sure you don't need anything before running this!

Quick Reference​

Create All Clusters Script​

For future reference, here's the complete process in one script (after discovering IPs once):

#!/bin/bash

# Create network
docker network create multicluster-example

# Create top cluster (replace IPs with yours)
k3d cluster create top \
  --api-port 6443 \
  --network multicluster-example \
  --k3s-arg "--cluster-domain=top.k3d.example.com@server:*" \
  --k3s-arg "--disable=traefik@server:*" \
  --k3s-arg "--tls-san=TOP_SERVERLB_IP@server:*" \
  --k3s-arg "--tls-san=TOP_SERVER_IP@server:*"

# Create mid cluster (replace IPs with yours)
k3d cluster create mid \
  --api-port 6444 \
  --network multicluster-example \
  --k3s-arg "--cluster-domain=mid.k3d.example.com@server:*" \
  --k3s-arg "--disable=traefik@server:*" \
  --k3s-arg "--tls-san=MID_SERVERLB_IP@server:*" \
  --k3s-arg "--tls-san=MID_SERVER_IP@server:*"

# Create bot cluster (replace IPs with yours)
k3d cluster create bot \
  --api-port 6445 \
  --network multicluster-example \
  --k3s-arg "--cluster-domain=bot.k3d.example.com@server:*" \
  --k3s-arg "--disable=traefik@server:*" \
  --k3s-arg "--tls-san=BOT_SERVERLB_IP@server:*" \
  --k3s-arg "--tls-san=BOT_SERVER_IP@server:*"

echo "All clusters created!"
k3d cluster list
